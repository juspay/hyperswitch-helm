# Project Brief

_This document serves as the foundation for the project, outlining its core requirements and goals. It is the source of truth for the project scope._

## Project Name

Hyperswitch (Helm Chart: `hyperswitch-app`)

## Core Goal

To provide a community-led, open payments switch that empowers digital businesses with fast, reliable, and affordable access to the best payments infrastructure.

## Key Features/Components (from initial observation)

*   **App Server:** The core application server handling payment logic.
*   **Control Center:** A UI for managing Hyperswitch, configuring processors, and testing payments.
*   **Card Vault:** A secure component for storing card information.
*   **Deployment:** Designed for Kubernetes using Helm charts.

## High-Level Requirements

*   Enable seamless integration with various payment processors.
*   Offer a robust and scalable payment processing solution.
*   Provide tools for monitoring and managing payment operations.
*   Ensure security of payment data, particularly card information.
*   Facilitate easy deployment and configuration on Kubernetes.
